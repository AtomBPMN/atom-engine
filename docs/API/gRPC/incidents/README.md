# Incidents Service

Сервис управления системными инцидентами в Atom Engine. Обеспечивает отслеживание, классификацию и решение ошибок в процессе выполнения BPMN.

## Обзор

Incidents Service предоставляет полный жизненный цикл управления инцидентами:
- Автоматическое создание при ошибках
- Классификация по типам и важности  
- Механизмы решения (retry/dismiss)
- Статистика и мониторинг
- Трассировка связанных объектов

## Доступные методы

### Управление инцидентами
- [**CreateIncident**](create-incident.md) - Создание нового инцидента
- [**ResolveIncident**](resolve-incident.md) - Решение инцидента (retry/dismiss)
- [**GetIncident**](get-incident.md) - Получение детальной информации

### Аналитика и мониторинг
- [**ListIncidents**](list-incidents.md) - Список инцидентов с фильтрацией
- [**GetIncidentStats**](get-incident-stats.md) - Статистика для мониторинга

## Типы инцидентов

- **JOB** - Ошибки при выполнении заданий
- **BPMN** - Ошибки в структуре процесса
- **EXPRESSION** - Ошибки в FEEL выражениях
- **PROCESS** - Ошибки процесса выполнения
- **TIMER** - Ошибки системы таймеров
- **MESSAGE** - Ошибки корреляции сообщений
- **SYSTEM** - Критические системные ошибки

## Статусы инцидентов

- **OPEN** - Новый инцидент, требует внимания
- **RESOLVED** - Решен через retry или исправление
- **DISMISSED** - Отклонен как несущественный

## Интеграция

Сервис автоматически интегрируется с:
- Process Service (для ошибок процессов)
- Jobs Service (для ошибок заданий)  
- TimeWheel Service (для ошибок таймеров)
- Messages Service (для ошибок корреляции)
- Expression Service (для ошибок вычислений)

## Рекомендации

- Регулярно проверяйте открытые инциденты
- Используйте статистику для выявления паттернов
- Настройте алерты на критические типы инцидентов
- Документируйте решения в комментариях
